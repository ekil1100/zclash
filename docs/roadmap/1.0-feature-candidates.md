# 1.0 功能候选清单（人话版）

> 供 Like 决策优先级。按 ROADMAP 1.0 退出条件对齐。

## 必须有（1.0 硬门槛）

| # | 功能 | 一句话描述 | 预估工作量 |
|---|------|-----------|-----------|
| 1 | CLI 命令模型稳定 | start/stop/status/proxy/profile 等核心命令语义冻结，`--json` 输出不再变 | 已完成 |
| 2 | API v1 资源模型锁定 | REST 6 类资源（runtime/profiles/proxies/connections/rules/metrics）+ 错误码表 | 已完成（文档） |
| 3 | 安装链路可用 | 新机器 3 步完成安装/验证/升级，失败给 next-step | 已完成 |
| 4 | 性能回归门禁 | 有基线、有阈值、有自动校验脚本，能挡住性能劣化 | 已完成（脚本级） |
| 5 | 稳定性基线 | 24h 长稳不崩、热重载可回滚、故障注入有恢复路径 | 脚本已有，需真实环境跑一轮（2-4h） |
| 6 | 配置迁移最小可用 | mihomo/clash 常见配置能自动检测 + 给修复建议（至少 port/log-level） | 已完成（2 条规则） |
| 7 | 72h 长稳通过 | 连续 72 小时无崩溃、无内存泄漏、无性能劣化 | 需真实运行窗口（72h） |
| 8 | 迁移边界文档 | 哪些能迁、哪些不能迁、不能迁的怎么绕——写清楚 | 已有框架，需补齐 2-3 个边界场景（2h） |

## 锦上添花（有了更好，没有不阻塞发布）

| # | 功能 | 一句话描述 | 预估工作量 |
|---|------|-----------|-----------|
| 1 | TUI 交互增强 | 代理组切换动画、连接筛选实时刷新、日志高亮 | 4-8h |
| 2 | 更多迁移规则 | 覆盖 proxy-groups/rules/dns 等高频配置字段 | 每条规则 1-2h |
| 3 | 对外摘要自动生成 | 试用结果一键导出可发送的简报 | 已完成 |
| 4 | 多平台安装包 | 除脚本安装外提供 brew/apt/docker 入口 | 8-16h |
| 5 | 诊断面板 | `zclash doctor` 一键输出环境/网络/配置健康度 | 4-6h |
| 6 | i18n 支持 | 错误提示和安装文案支持中英文切换 | 框架已有，补翻译 2-4h |

## 与 1.0 退出条件的对齐

| ROADMAP 1.0 退出条件 | 对应必须有项 |
|---|---|
| 连续两个迭代无 P0/P1 未收敛 | #1 CLI + #2 API + #4 回归门禁 |
| 72h 长稳通过 | #5 + #7 |
| 迁移边界清晰 | #6 + #8 |
| 运维与诊断能力可支持稳定对外使用 | #3 安装链路 + #5 稳定性 |
